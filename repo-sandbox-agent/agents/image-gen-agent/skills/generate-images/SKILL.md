# Generate Images Skill

## Purpose
Receive an image generation spec from the architect, craft precise prompts for each
layout zone, and output a complete config consumed by the site runtime. Images are
served through a three-tier stack; this skill writes both the config and the prompts.

## Inputs
Expect a JSON spec from the architect:
```json
{
  "theme": "dark 1980s supernatural small town",
  "vibe": "horror, nostalgic, neon",
  "images": [
    {
      "zone": "hero",
      "prompt_intent": "atmospheric aerial view of a small American town at night, foggy streets",
      "width": 1920,
      "height": 1080,
      "style": "cinematic, dark, photorealistic"
    },
    {
      "zone": "card",
      "prompt_intent": "retro diner neon sign at night",
      "width": 800,
      "height": 600,
      "style": "cinematic, nostalgic"
    }
  ],
  "animations": ["flicker", "float", "glitch-text", "fade-in-up", "red-pulse"]
}
```

## Step 1 — Craft Prompts
For each image in `spec.images`:
1. Combine `prompt_intent` + `spec.vibe` keywords + `image.style` into one prompt
2. Keep under 200 characters
3. Use ASCII only — no quotes, apostrophes, or special chars (breaks URL encoding)
4. Append style direction: `cinematic`, `photorealistic`, `digital art`, `concept art`, etc.
5. Compute seed: `theme.charCodeAt(0) * 1000 + theme.length * 17`

## Step 2 — Build Three-Tier Config
For each image zone produce three entries:

### Tier 1 — Gemini Imagen (primary)
```json
{
  "tier": "gemini",
  "model": "imagen-3.0-generate-001",
  "prompt": "<crafted prompt>",
  "width": 1920,
  "height": 1080,
  "zone": "hero"
}
```

### Tier 2 — Pollinations.ai (fallback)
Build the URL directly:
```
https://image.pollinations.ai/prompt/{url-encoded-prompt}?width=W&height=H&nologo=true&seed=N
```
```json
{
  "tier": "pollinations",
  "url": "https://image.pollinations.ai/prompt/dark+atmospheric+town+at+night+foggy+cinematic?width=1920&height=1080&nologo=true&seed=7734",
  "zone": "hero",
  "prompt": "dark atmospheric town at night foggy cinematic",
  "alt": "Dark atmospheric aerial view of a foggy town at night"
}
```

### Tier 3 — Puter.js (client-side)
Emit the prompt into `puter-image-config.js` so `PuterImageLoader.tsx` can call
`puter.ai.txt2img(prompt)` in the browser:
```json
{
  "tier": "puter",
  "prompt": "dark atmospheric town at night foggy cinematic",
  "zone": "hero",
  "alt": "Dark atmospheric aerial view of a foggy town at night"
}
```

## Step 3 — Write puter-image-config.js
Write the full three-tier config to `generated-site/public/puter-image-config.js`:
```js
// Auto-generated by image-gen-agent — do not edit manually
window.PUTER_IMAGE_CONFIG = {
  images: [
    {
      zone: "hero",
      alt: "Dark atmospheric aerial view of a foggy town at night",
      gemini: {
        model: "imagen-3.0-generate-001",
        prompt: "dark atmospheric small town at night foggy streets vintage cars glowing streetlights cinematic horror 1980s"
      },
      pollinations: {
        url: "https://image.pollinations.ai/prompt/dark+atmospheric+small+town+at+night+foggy+streets+vintage+cars+cinematic+horror+1980s?width=1920&height=1080&nologo=true&seed=7734"
      },
      puter: {
        prompt: "dark atmospheric small town at night foggy streets vintage cars glowing streetlights cinematic horror 1980s"
      },
      width: 1920,
      height: 1080
    }
  ]
};
```

## Step 4 — Write CSS Animations
For each animation name in `spec.animations`, produce a complete @keyframes block
with a usage class and a `prefers-reduced-motion` override.

Standard animation library:
- `flicker`: neon light flicker for horror/dark themes
- `float`: gentle vertical float for badges and hero elements
- `glitch-text`: digital RGB glitch for dark/tech hero titles
- `fade-in-up`: smooth entry for cards and sections
- `red-pulse`: danger/horror glow pulse for CTAs
- `scan-lines`: CRT overlay for retro/horror backgrounds
- `neon-glow`: pulsing neon border/text glow for sci-fi
- `sway`: gentle side-to-side for organic/nature themes
- `slide-in`: clean horizontal entry for corporate layouts
- `data-stream`: scrolling matrix-style characters for tech themes

## Output Contract
Return exactly this JSON structure (no markdown):

## Prompt Engineering Rules
1. Describe aesthetics, not characters: "tall shadowy figure in 1980s suburb" not "Demogorgon"
2. Always include: mood · lighting · era · composition type
3. Pollinations seed: use consistent seed per site for reproducibility
4. nologo=true: always append to suppress Pollinations watermark
5. Max 4 AI-generated images per site (performance budget)
6. For Gemini: use `imagen-3.0-generate-001` model
7. For Puter: prompts go to `puter-image-config.js`, never called directly from this agent
